
협업하는 과정 3단계!
1단계 : 누가 이 작업을 할 것인지 정하기 - 이슈 할당
2단계 : 각자 맡은 것을 작업 - 브랜치 생성
3단계 : 각자 작업물을 한 프로젝트로 합친다 - 서로 작업한 내용을 리뷰(PR)하고 최종적으로 반영 : 머지!


2주차 주요 개념 키워드 적어보기

Issue ; 내가 할 작업, 기능 추가, 버그 리포트 등 여러 방식으로 사용이 가능하다.
Branch : 특정 커밋에서 나뭇가지처럼 갈라져 작업할 수 있다.
브랜치명은 규칙을 가지고 이름을 잘 지어놓으면 프로젝트 관리가 쉽다.
Checkout : 작업할 브랜치로 전환하는 것
Merge : 브랜치의 작업 내역 커밋들을 합치는 것
기준이 되는 기본 브랜치를 정하고 이 브랜치에 내용을 모두 머지!
Merge Conflict : 머지하는 과정에서 같은 파일에서 동일한 부분이 수정된 게 발견되면 발생하는 것
충돌을 수정하려면 최종적으로 반영할 내역으로 고친 후 Merge commit 진행하면 끝~!
Tracking : 로컬 repo와 원격 repo의 특정 브랜치를 연결해주는 것
push와 pull은 기본적으로 트래킹되고 있는 브랜치를 기준으로 커밋 내역을 반영한다.
